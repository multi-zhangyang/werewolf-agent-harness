## 一、多 Agent 对抗 / 狼人杀 LLM 技术 (2024-2025)

### 1. WOLF (arXiv:2512.09187, NeurIPS 2025 MTI-LLM Spotlight)
多 agent 狼人杀欺骗基准,7320 条陈述。
- **欺骗四分类**:omission(遗漏)/distortion(扭曲)/fabrication(捏造)/misdirection(误导)。结构化分析"如何骗"而非"是否骗"。
- **生产 vs 检测分离**:说话者自评诚实度,他人评欺骗度,独立测量。
- **纵向怀疑平滑**:怀疑度跨轮平滑,捕捉演化信任。狼人怀疑从 ~52% 升到 >60%,村民/医生稳定 ~44-46%。
- **误报控制**:多轮交互提升对狼的召回,且**不累积误判诚实角色**(村民怀疑稳定不暴涨)——这是关键发现。
- **结果**:狼人 31% 回合产出欺骗;peer 检测 71-73% precision,~52% overall accuracy。
- LangGraph 状态机强制 night-day 循环 + 辩论 + 多数投票。

### 2. Sato "不完全信任 LLM" (arXiv:2409.01575)
hybrid LLM + 规则选择。
- LLM 做对话分析 + 候选发言生成;**规则算法在 LLM 输出与预制模板间选择**。
- 解决三个问题:情境反驳、会话终止判断、人设一致性。
- 纯 LLM 会产生逻辑矛盾、幻觉游戏状态、不知何时结束对话。规则层 gatekeeping。
- 启示:我们的 `_sanitize_*` 已有就近修正,但缺**会话终止检测**(bid=0 收敛部分覆盖)和**模板兜底**。

### 3. MultiMind (arXiv:2504.18039, ACMMM 2025)
首个多模态 SDG agent,ONUW 测试。
- **递归 Theory of Mind**:不只推断他人身份,还建模"他人如何看待自己/同伴"——**A 认为 B 怎么看 C**。
- ToM + MCTS:搜索使"对自己的怀疑最小化"的沟通策略。
- 启示:我们只有一阶 suspicion(seat→trust),缺**二阶信任网络**。

### 4. ONUW 策略性讨论 (arXiv:2405.19946)
- **Perfect Bayesian Equilibrium** 存在;**讨论通过改变玩家信念大幅改变效用**——讨论不是调味,是核心策略。
- RL 训练讨论策略:决定"采用哪种讨论战术",而非自由生成。
- 启示:可加**讨论战术选择器**(指控/辩护/拉拢/澄清/转移),而非让 LLM 自由发挥。

### 5. GRATR 证据图 (arXiv:2408.12333)
零样本图检索增强信任推理。
- **观察期**:观察他人行动→构建信任图(边=信任关系,动态演化)。
- **决策期**:多跳检索评估对特定目标的信任,从多个可信源拉证据链。
- 结果:狼人游戏推理准确率 +50.5%,幻觉 -30.6%。
- 启示:可加**claim 矛盾检测图**——跨轮追踪每人 claim,检测前后矛盾(狼人易前后不一)。

### 6. 其他
- **WereWolf-Plus** (2506.12841):多模型多维度基准,可定制角色/模型分配。
- **Verbal Werewolf** (2506.00160):fine-tuned TTS 近实时语音狼人杀。
- **LSPO** (2502.04686, ICML 2025):自由发言→潜策略空间,CFR+DPO 微调。
- **DVM** (2501.06695):可控胜率 RL reward。
- **Enhance Reasoning** (2402.02330):外部 Thinker 模块做 System-2 推理,6B+Thinker 超 GPT-4。
- **Helmsman** (2404.01602, COLM 2024):警长评估 LLM 意见领导力,多数 LLM 不具备。
- **Beyond Survival** (2510.11389):100h 多模态狼人数据集 32.4M token,策略对齐评估,SOTA 约 half <0.50。
- **CSP4SDG** (2511.06175):概率约束满足做角色推断,超 LLM baseline。
- **TextArena** (2504.11442):57+ 竞争文本游戏,评 negotiation/ToM/deception。

## 二、狼人杀游戏设计要点(综合)
- 信息隔离:每人只看自己的私有信息 + 公开事件(我们已做 `AgentObservation` 投影)。
- 纵向信任:跨轮平滑,避免单轮误判暴涨(我们已做 `smoothing=0.35`)。
- 欺骗要有策略性选用,不是每句都骗(WOLF + 我们的 wolf prompt)。
- 讨论是核心策略载体,通过改变信念改变效用(ONUW PBE)。
- 规则层兜底防 LLM 幻觉/矛盾(Sato)。
- 评估:precision/recall/纵向怀疑曲线/策略对齐(Beyond Survival)。

## 三、UI 设计(狼人杀/社交推理游戏)
WebSearch 不可用,基于训练知识 + 论文系统描述:
- **座位布局**:圆桌径向(Wolvesville/Town of Salem 2)或卡片网格(移动优先)。Blood on the Clocktower 用 token 极简风。
- **暗色主题**:深海军蓝/炭灰底(#1a1d29/#0f1117)减长局眼疲劳;阵营色——狼=深红、村民=金、中立=紫。
- **状态可视化**:发光边框/阴影表示当前发言者、投票状态、夜间行动;glassmorphism 面板放行动日志/聊天。
- **技术栈**:React/Next.js + Tailwind;Framer Motion 座位过渡/角色揭示动画;WebSocket 实时;Canvas/SVG 圆桌。
- 参考灵感:Wolvesville、Town of Salem 2、Blood on the Clocktower、Mafia.gg、Dribbble "mafia game UI"。

## 四、落地到 werewolf-mas 的改进项(按杠杆排序)
1. **二阶 ToM(递归信任网络)** — 不只 seat→suspicion,还建模"A认为B怎么看C"。MultiMind 核心技术,我们完全没有。
2. **claim 矛盾检测图** — 跨轮追踪每人 claim,检测前后矛盾(GRATR 思路)。狼人易前后不一,这是硬信号。
3. **讨论战术选择器** — 指控/辩护/拉拢/澄清/转移,RL-like 选择(ONUW)。先做规则版。
4. **村民怀疑度钳制** — WOLF 误报控制:村民怀疑度不应单轮暴涨,加上限。
5. **会话终止检测** — Sato:连续 bid=0 或无新信息时果断结束(我们 bid=0 收敛已部分覆盖)。
6. **对局质量自动评分** — TextArena/Beyond Survival:precision/recall/策略对齐,跑完自动出报告。
7. **UI**:圆桌径向座位 + 发光当前发言者 + glassmorphism 日志面板 + 角色揭示动画(当前是网格,可升级)。
